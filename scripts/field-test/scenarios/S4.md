# S4 — refactor & lifecycle via the guard + `pull` reconciliation

**Persona.** Same user (Priya), tidying up. Casual, outcome-focused wording —
"clean this up", "we don't need the old one anymore", "give me types on that
step so I stop fat-fingering fields". She does **not** know decanter's verbs;
she describes intent.

**What's under test (the #107 reconciliation machinery — S4's most valuable
role).** The structure/lifecycle verbs are **retired**: rename, create, archive,
node-rename are **n8n MCP acts through the guard**, and `pull` reconciles the
local mirror. S4 drives exactly that:

1. **Rename a node** and **rename the workflow** over MCP through the guard
   (`renameNode` / `setWorkflowMetadata`) →
2. **`pull` reconciles**: the local `code/` file **follows the node rename**,
   the cached workflow `name` **re-caches**, and the folder slug **stays sticky**
   (never follows a remote rename). With `liveMirror` on, the tracked
   workflow's mirror may refresh in the background before the agent's explicit
   `pull`.
3. **Archive an obsolete seeded workflow** over MCP (`archive_workflow`) — it
   survives server-side (`isArchived`), local folder untouched as the git
   record; a later `pull` of it is refused ("is archived and cannot be
   accessed").
4. **Convert one Code node to TypeScript** — there is **no `--ts` verb**: the
   agent **replaces `code/<node>.js` with `code/<node>.ts`, re-points the
   `//@file:` placeholder in `workflow.json` to the `.ts` file, and `push`es**.
   Push compiles via esbuild and appends the `// @ts-n8n sha256:<hash>` marker.

## Beats (adaptive)

- **Agent reaches for a retired verb** (`n8n-decanter rename`, `create`,
  `archive`, `node rename`). This is **signal** — log the exact command. It will
  fail as unknown; watch whether the agent recovers to the MCP/guard path.
- **Agent reaches for `backup`** under "we don't need the old import flow
  anymore" wording. Also signal (Plan 35 logs it) — `backup` is git-native
  disaster recovery, orthogonal to archiving. Log it; don't correct.
- **TS conversion confusion.** If the agent expects a decanter verb to convert,
  or forgets to re-point the placeholder, note it. The correct path is
  file-swap + placeholder edit + push.
- **Casual archive wording (turn 2).** "we don't run the old import flow
  anymore — get rid of it so it stops cluttering the list." (Which seeded
  workflow is "the old import flow" is in the stage manifest.)

## Success checklist (grader + `verify.mts`)

- Node rename + workflow rename done via **MCP through the guard**; a
  subsequent `pull` **reconciled** — file followed the node rename, `name`
  re-cached in `.decanter.json`, folder slug unchanged (sticky).
- Archive done via **MCP** (`archive_workflow`); the seeded workflow is
  `isArchived` server-side; its local folder is intact.
- TS conversion: `verify.mts` checks the converted node via the **marker-hash
  relation** (remote = compiled JS + valid `@ts-n8n` marker), **NOT**
  byte-equality to the `.ts` source. `.ts` file present; placeholder re-pointed.
- **No decanter rename/create/archive/node-rename verb succeeded** (they don't
  exist) — any attempt is logged as a finding.
- `verify.mts` green for every still-tracked workflow.
