# S5 (optional, unsandboxed only) — watch loop

> **Draft / defer if flaky.** `fs.watch` (macOS FSEvents) dies under sandboxed
> shells; run only unsandboxed. Optional for round 1.

**Persona.** Same user (Priya). "I want my edits to just show up in n8n while I
work" — she wants a tight local edit → live loop.

**What changed (Plan 52 / #128 — the browser-reload proxy is GONE).** `watch` no
longer serves a reload proxy. It **pushes on save** (code files → draft) and
prints the **editor deep link**; the "just show up" reflection is now **n8n's
native** behaviour — the editor soft-re-renders on external MCP draft edits. So
"just show up in n8n" is **n8n's** behaviour to observe, **not a decanter
surface to grade**. S5 grades the *watch push loop*, not any reload proxy.

**Goal.** Start `watch` on a workflow, do a few edit → save → (auto)push cycles,
confirm each save reaches the **draft** and the workflow keeps executing after a
`publish`. Observe (don't grade) that the open n8n editor reflects the draft.

## Beats (adaptive)

- **Agent looks for a reload proxy / preview URL.** It won't find one (removed).
  Note if the agent expects it (stale mental model) — but this is observation,
  not a decanter defect.
- **Save didn't push.** 200 ms debounce; allow ~500 ms after a write before
  concluding. A `workflow.json` save **warns and pushes nothing** (read-only
  snapshot) — that's correct.
- **No git.** `watch` refuses its startup pull without git and warns "no git
  safety net" — the scratch dir is a repo by S5, so this shouldn't fire.

## Success checklist (grader)

- `watch` started; each **code** save auto-pushed to the **draft** (live
  unchanged until `publish`); a `workflow.json` save warned and pushed nothing.
- Agent used the printed **editor deep link** rather than hunting a (removed)
  reload proxy.
- Post-`publish`, the workflow still executes.
- `verify.mts` byte-equality holds across the edit cycles.
