# S1 — green field, simple (CLI-only)

**Persona.** Priya, a solo operations analyst. Just spun up her own n8n, minted
an MCP token in the UI (Settings → MCP → API key). She keeps her flows in a git
folder and wants one small workflow finished. Comfortable in a terminal, not an
n8n expert.

**Design decision (per Plan 35 — "decide and state which").** S1 is
**CLI-only against a stage-seeded skeleton**. The stage pre-creates a
manual-trigger → **empty Code node** workflow (`availableInMCP = true`) so S1
exercises `init` → `pull` → author → verify → `push` → `publish` **without**
depending on the `mcp connect` guard being loaded (it can't be, in the very
session that `init` first writes `.mcp.json`). The MCP-guard creation path is
S2's job, in a later session where `.mcp.json` already exists.

**Goal.** Set up the sync dir with `init` (MCP token **paste** path — no browser
consent; that's e2e's job), pull the skeleton, write the Code node to dedupe and
normalise a pasted JSON list, confirm it works locally, then push and publish.

## Beats (adaptive — send only if the condition fires)

- **init prompts stall.** `init` asks for host, then MCP token, then an optional
  API key over stdin. If the agent hangs or can't get past the prompts, nudge:
  "it asked me for the host and a token — I pasted the token from n8n, the API
  key is optional." (The token + host + api key are in the goal prompt already.)
- **Agent asks how to verify before shipping.** Answer in character: "yeah I'd
  rather not flip it live until I've seen it do the right thing on a sample."
  (Watch whether it reaches for `preflight`, `node run`, `check`, or `test` —
  record which; do not name a verb for it.)
- **Change of mind (send as turn 2 regardless).** "oh — and drop any rows where
  `status` is `archived`, we don't care about those."
- **Agent tries to create the workflow via MCP in turn 1.** Fine, but note it:
  in turn 1 `.mcp.json` was just written by `init`, so the guard is not loaded
  in that process. If it fails to reach the instance over MCP mid-turn-1, that's
  expected — the skeleton is already there to `pull`.

## Success checklist (grader + `verify.mts`)

- `init` completed via the **token-paste** path; wrote `.env` (host + MCP token,
  api key optional) and scaffolded the template (`.mcp.json`, `AGENTS.md`,
  `.claude/`, `decanter.config.json`, tsconfig, `n8n-globals.d.ts`).
- `pull` landed the skeleton as `workflows/<slug>/` with an **empty** `code/`
  file for the Code node (born without `jsCode`).
- The Code node source was authored **in the file**, not via MCP; dedupe +
  normalise + (turn 2) drop-archived logic present.
- Some **local verification** ran before publish — `preflight`, `node run` with
  a fixture, `check`, or `test`. **Record which** (this is the graded
  preflight-adoption question).
- `push` reported a **draft** update (live unchanged); `publish` took it live.
- `verify.mts`: remote `jsCode` **byte-equals** the local `.js`; workflow.json
  placeholder intact; `.decanter.json` only via `decanter:` auto-commits.
